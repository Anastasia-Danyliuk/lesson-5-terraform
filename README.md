Цей проєкт містить інфраструктуру AWS, розгорнуту за допомогою Terraform. Структура побудована з використанням модулів для забезпечення повторного використання та зручності управління.

## Структура проєкту
```text
lesson-5/
├── main.tf                  # Головний файл для підключення модулів
├── backend.tf               # Налаштування бекенду для стейтів (S3 + DynamoDB)
├── outputs.tf               # Загальне виведення ресурсів
├── modules/                 # Каталог з усіма модулями
│   ├── s3-backend/          # Модуль для S3 та DynamoDB
│   ├── vpc/                 # Модуль для VPC
│   └── ecr/                 # Модуль для ECR
└── README.md                # Документація проєкту
```
## Опис модулів
s3-backend: Налаштовує S3-бакет (з версіюванням та шифруванням AES256) для зберігання стейт-файлів та DynamoDB-таблицю для блокування стану.

vpc: Створює мережеву інфраструктуру: VPC, 3 публічні та 3 приватні підмережі, Internet Gateway та NAT Gateway з відповідними таблицями маршрутизації.

ecr: Створює репозиторій для Docker-образів з налаштуваннями сканування образів та політикою доступу.


## Команди для роботи
# Для ініціалізації та розгортання інфраструктури використовуйте наступні команди:
# Ініціалізація проєкту:
Bash
terraform init
# Перегляд плану змін:
Bash
terraform plan
# Застосування конфігурації:
Bash
terraform apply
# Видалення інфраструктури:
Bash
terraform destroy
## Вимоги до налаштування
Проєкт налаштовано з використанням required_providers (AWS). Всі ресурси мають відповідне тегування та конфігурації безпеки (шифрування S3/ECR, DNS-налаштування VPC).
